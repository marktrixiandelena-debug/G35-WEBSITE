# WIREFRAME CORE COMPONENTS (LOW-FI vs. HI-FI)

This document outlines the core components included in both the **Low-Fidelity (Low-Fi)** and **High-Fidelity (Hi-Fi)** designs of the **Barangay Calauag Flood and Drainage Reporting System**. This approach ensures that the system's structure is solid before visual branding is applied.

---

## 1. LOW-FIDELITY (LOW-FI) CORE COMPONENTS
The focus of the Low-Fi wireframes is **Structure, Usability, and Layout**. It serves as the skeletal framework of the application.

*   **Layout & Grids:** Defines where major elements (Sidebar, Header, Content Area) sit on the screen using basic grayscale containers.
*   **Navigation Hierarchy:** Shows the logical flow between pages (e.g., clicking a sidebar link to change the content view) without the distraction of colors.
*   **Functional Placeholders:**
    *   **Images:** Boxes with an "X" to represent where flood evidence photos will be displayed.
    *   **Icons:** Simple circles or squares representing buttons like "Profile" or "Logout."
*   **UI Controls (Inputs/Buttons):** Basic outlines for text fields, dropdowns (Incident Type/Severity), and buttons to test the "Click Flow."
*   **Typography Hierarchy:** Uses generic fonts to show which text is a heading (H1, H2) and which is body text, focusing on information priority.

---

## 2. HIGH-FIDELITY (HI-FI) CORE COMPONENTS
The focus of the Hi-Fi mockups is **Visual Identity, Branding, and User Experience (UX) Detail**. This is the final look of the website.

*   **Brand Identity (Teal & Emerald):** Application of the official color palette (#1F7A8C Primary Teal, #065F46 Emerald Green) to all surfaces and buttons.
*   **Typography (Inter Font):** Implementation of the refined `Inter` font family with specific weights (Bold for numbers, Semibold for headers) to improve emergency readability.
*   **Visual Assets & Icons:** 
    *   **High-Quality Icons:** Using specific FontAwesome or Custom icons for floods, teams, and alerts.
    *   **Realistic Content:** Using actual sample images of flooding and localized city data instead of "Lorem Ipsum."
*   **Micro-Interactions & Polish:**
    *   **Interactive States:** Inclusion of hover effects (buttons lifting), focus states (green border on inputs), and active sidebar indicators.
    *   **UI Detail:** Addition of shadows for depth (Elevations), 16px border-radius for cards, and glassmorphism (back-drop blur) on the header.
*   **Status Indicators (Semantic Colors):** Visual differentiation of report statuses using specific colors (Red for Critical, Green for Resolved) for instant scanning.

---

## COMPARISON SUMMARY

| Component | Low-Fi Focus | Hi-Fi Focus |
| :--- | :--- | :--- |
| **Colors** | Black, White, & Gray | Full Brand Palette (Teal, Emerald, Orange) |
| **Images** | Placeholder Boxes with 'X' | High-Resolution Incident Photos |
| **Content** | Lorem Ipsum / Generic Labels | Real Localized Data |
| **Buttons** | Rectangular Outlines | Gradients, Shadows, & Hover States |
| **Purpose** | To define **Functionality** | To define **Aesthetics & Branding** |
