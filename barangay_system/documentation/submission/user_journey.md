# USER FLOW AND JOURNEY MAP

## USER FLOW
A user flow is a step-by-step sequence that outlines how a user interacts with a product or service. It provides a structured view of the user's actions and the system's responses. For the **Barangay Calauag Flood and Drainage Reporting System**, the user flow starts with actions like logging in or registering for an account. It progresses through critical steps like selecting the incident type, providing detailed location information, and uploading photographic evidence of the flooding or drainage issue. The flow continues as the user submits the report and monitors its real-time status as it moves from "Pending" to "Resolved." It concludes with the resident receiving confirmation of the resolution and potentially viewing the final actions taken by the barangay response team.

### Primary Resident Flow:
1.  **Authentication:** Login or Register for a resident account.
2.  **Dashboard:** Access the main dashboard to see current alerts.
3.  **Report Initiation:** Click on "Submit New Report."
4.  **Data Entry:** Input incident type (Flood/Drainage), severity, and location details.
5.  **Evidence Upload:** Attach photos of the incident.
6.  **Submission:** Confirm and submit the report to the Barangay database.
7.  **Tracking:** Monitor the "My Reports" page for live status updates.
8.  **Resolution:** View the final resolution details once the barangay team completes the task.

## JOURNEY MAP
The User Journey Map visualizes the resident's experience and emotional state while interacting with the system during a flooding event.

| Phase | Action | Emotional State | System Touchpoint |
| :--- | :--- | :--- | :--- |
| **Awareness** | Resident notices rising water or a clogged drainage. | Concerned / Anxious | Landing Page |
| **Reporting** | Residents logs in and fills out the incident form. | Proactive / Hopeful | Submit Report Page |
| **Evidence** | Resident takes a photo of the flood and uploads it. | Empowered | Photo Upload Component |
| **Waiting** | Resident checks the status of the report periodically. | Expectant | My Reports / Tracking Page |
| **Action** | Barangay dispatches a team (Status: In Progress). | Reassured | Real-time Status Notification |
| **Resolution** | Team resolves the issue (Status: Resolved). | Relieved / Satisfied | Resolution Summary |
